# Terraform Infrastructure for Prosper Tutor Lite\n\nThis directory contains Terraform configurations to provision cloud infrastructure for deploying Prosper Tutor Lite in production.\n\n## Overview\n\nThe Terraform code provisions:\n- A VPC with public and private subnets\n- An EKS (Elastic Kubernetes Service) cluster\n- Managed node groups for compute capacity\n- Supporting IAM roles and security groups\n\n## Prerequisites\n\n- Terraform >= 1.0\n- AWS CLI configured with appropriate credentials\n- kubectl for interacting with the Kubernetes cluster\n\n## Quick Start\n\n1. Initialize Terraform:\n   ```bash\n   terraform init\n   ```\n\n2. Review the execution plan:\n   ```bash\n   terraform plan -var-file=\"terraform.tfvars\"\n   ```\n\n3. Apply the infrastructure:\n   ```bash\n   terraform apply -var-file=\"terraform.tfvars\"\n   ```\n\n4. Configure kubectl to interact with the cluster:\n   ```bash\n   aws eks update-kubeconfig --name $(terraform output -raw cluster_name) --region $(terraform output -raw aws_region)\n   ```\n\n## Modules Used\n\n- [VPC Module](https://registry.terraform.io/modules/terraform-aws-modules/vpc/aws)\n- [EKS Module](https://registry.terraform.io/modules/terraform-aws-modules/eks/aws)\n\n## Variables\n\nRefer to `variables.tf` for a complete list of configurable variables.\n\n## Outputs\n\nAfter applying, Terraform will output important information such as:\n- Cluster endpoint for accessing Kubernetes API\n- Command to configure kubectl\n- VPC and subnet IDs for reference\n\n## Cleanup\n\nTo destroy the provisioned infrastructure:\n```bash\nterraform destroy -var-file=\"terraform.tfvars\"\n```\n\nNote: This will permanently delete all resources created by Terraform."
